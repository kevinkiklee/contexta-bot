# Message Log Overhaul

## Problem

The dashboard history page shows messages from a 50-message Redis rolling window per channel. No search, no filtering, no sorting, no long-term persistence. Messages are plain text strings with no structured metadata.

## Solution

Add a `messages` table in PostgreSQL for durable, searchable message storage. The bot writes to both Redis (LLM context) and Postgres (dashboard). The dashboard reads from Postgres with full-text search, semantic search, filtering, sorting, and cursor-based pagination.

## Schema

```sql
CREATE TABLE messages (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  server_id     varchar(255) NOT NULL,
  channel_id    varchar(255) NOT NULL,
  user_id       varchar(255) NOT NULL,
  display_name  varchar(255) NOT NULL,
  content       text NOT NULL,
  is_bot        boolean NOT NULL DEFAULT false,
  created_at    timestamptz NOT NULL DEFAULT now(),
  embedding     vector(768),
  search_vec    tsvector GENERATED ALWAYS AS (to_tsvector('english', content)) STORED
);

CREATE INDEX idx_messages_channel_time ON messages (server_id, channel_id, created_at DESC);
CREATE INDEX idx_messages_search ON messages USING GIN (search_vec);
CREATE INDEX idx_messages_embedding ON messages USING hnsw (embedding vector_cosine_ops);
CREATE INDEX idx_messages_user ON messages (server_id, user_id);
```

## Write Path

1. Bot `messageCreate` event: write to Redis (unchanged) + POST to backend `/api/messages`
2. Bot reply: also POST to `/api/messages` with `isBot: true`
3. Backend inserts into `messages` table. `search_vec` auto-generated by Postgres.
4. Embedding worker backfills `embedding IS NULL` rows asynchronously.

## API

`GET /api/messages?serverId=X`

Query params:
- `serverId` (required)
- `channelId` (optional)
- `q` (optional — search query)
- `searchMode` — `text` (default) or `semantic`
- `userId` (optional)
- `botOnly` (optional)
- `before` (cursor — ISO timestamp for pagination)
- `limit` (default 50, max 100)

Returns: `{ messages: Message[], nextCursor: string | null }`

## Dashboard UI

Single unified view at `/dashboard/[serverId]/history`:
- Search bar with text/semantic toggle
- Filter row: channel dropdown, user dropdown, bot-only toggle
- Message list with cursor-based "Load more" pagination
- Sidebar channel list pre-fills channel filter on click
